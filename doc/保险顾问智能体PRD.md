# 保险顾问智能体 PRD

## 1. 产品定位

**定位：** 嵌入式保险顾问助手，为用户提供百万医疗险和重疾险的专业问题解答。

**边界限定：**
- ✅ 解答险种条款疑问
- ✅ 解释保障责任范围
- ✅ 说明责任免除要点
- ✅ 帮助理解专业术语
- ❌ 不提供购买建议
- ❌ 不推荐具体产品
- ❌ 不进行产品对比（对比功能由主流程承担）

## 3. 交互规则

| 规则 | 描述 |
|------|------|
| 页面状态保持 | 打开智能体对话页时，原页面（如对比结果页）**不关闭、不遮挡** |
| 布局模式 | 右侧抽屉式滑入，占比 380px 宽度，全高度 |
| 对话历史 | 仅在当前项目tab存活，切换tab保留，关闭项目清除 |
| 历史持久化 | 存储于 LocalStorage，key 包含 tab 标识 |
| 多会话隔离 | 不同项目tab独立存储，互不影响 |

## 4. 智能体对话逻辑

### 4.1 话题边界控制

| 用户输入场景 | 系统行为 |
|-------------|---------|
| 询问百万医疗险/重疾险相关 | 正常解答 |
| 询问其他险种（寿险/意外险等） | 礼貌拒绝并引导："抱歉，我仅支持解答百万医疗险和重疾险相关问题" |
| 询问产品推荐/对比 | 礼貌拒绝："我无法提供产品推荐，建议您使用对比功能，或咨询专业顾问" |
| 询问购买/投保流程 | 礼貌拒绝："具体投保流程建议咨询保险公司官方渠道" |
| 敏感词触发 | 统一引导至正规咨询渠道 |

### 4.2 对话流程

```
用户输入 → 话题边界校验
    ├─ 合规 → 调用 LLM 解答 → 追加历史记录 → 渲染消息
    └─ 超范围 → 返回引导话术 → 追加历史记录 → 渲染消息
```

### 4.3 快捷问题（初始显示）

| 序号 | 问题 |
|------|------|
| 1 | 百万医疗险的免赔额是什么意思？ |
| 2 | 重疾险和百万医疗险有什么区别？ |
| 3 | 等待期出险会怎样？ |
| 4 | 什么情况下百万医疗险不报销？ |

## 5. 数据结构

### 5.1 对话消息结构

```typescript
interface Message {
  id: string;           // UUID
  role: 'user' | 'assistant';
  content: string;
  timestamp: number;    // 时间戳
  status: 'sent' | 'error';  // 发送状态
}

interface ConversationStore {
  tabId: string;        // 当前项目tab标识
  messages: Message[];
  createdAt: number;
  updatedAt: number;
}
```

### 5.2 LocalStorage 存储结构

```json
{
  "insurance_advisor_conversations": {
    "tab_project_abc123": {
      "messages": [...],
      "createdAt": 1748915200000,
      "updatedAt": 1748915600000
    }
  }
}
```

### 5.3 对话清除机制

| 事件 | 清除行为 |
|------|---------|
| 关闭当前项目tab | 清除该tab对应的对话历史 |
| 关闭浏览器/刷新页面 | 清除所有对话历史 |
| 切换至其他项目tab | 保留当前tab对话，加载目标tab历史（如有） |

## 6. 技术实现

### 6.1 前端组件

| 组件 | 职责 |
|------|------|
| `AdvisorFloatButton` | 悬浮入口按钮 |
| `AdvisorDrawer` | 右侧抽屉容器 |
| `ConversationList` | 消息列表渲染 |
| `MessageBubble` | 单条消息气泡 |
| `QuickQuestions` | 快捷问题卡片区 |
| `ConversationInput` | 输入框+发送 |

### 6.2 状态管理

| 状态 | 存储位置 | 说明 |
|------|---------|------|
| 对话列表 | LocalStorage | 持久化，跨刷新保留 |
| 当前输入 | 组件内 State | 仅内存 |
| 抽屉开合状态 | 组件内 State | 仅内存 |
| loading 状态 | 组件内 State | 仅内存 |

### 6.3 后端接口

**POST /api/advisor/chat**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| question | string | 是 | 用户问题 |
| context | string | 否 | 当前方案信息上下文 |

**返回：**
```json
{
  "code": 200,
  "data": {
    "answer": "string",
    "topicScope": "in_scope" | "out_of_scope",
    "redirectMessage": "string | null"
  }
}
```

## 8. 性能与兼容性

| 指标 | 要求 |
|------|------|
| 首屏加载 | 智能体组件懒加载，不影响主流程 |
| 对话响应 | ≤3s（LLM 接口超时 10s） |
| 消息数量上限 | 单会话最多保留 50 条，超出自动折叠早期消息 |
| 存储容量 | LocalStorage 单 key ≤500KB |

## 9. 风险管理

| 风险 | 预案 |
|------|------|
| LLM 接口超时 | 显示"当前咨询人数较多，请稍后再试" |
| 敏感话题绕过 | 后端增加话题检测层，强制拦截 |
| 存储满 | 清理最早会话，保留最近 3 个 tab 的对话 |
| 用户误删对话 | 不提供恢复功能，引导重新开始 |
